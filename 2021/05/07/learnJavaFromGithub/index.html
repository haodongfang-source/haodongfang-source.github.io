从Java 14开始，switch语句正式升级为表达式，不再需要break，并且允许使用yield返回值。 

public class Main {
    public static void main(String[] args) {
        String fruit = "apple";
        switch (fruit) {
        case "apple" -> System.out.println("Selected apple");
        case "pear" -> System.out.println("Selected pear");
        case "mango" -> {
            System.out.println("Selected mango");
            System.out.println("Good choice!");
        }
        default -> System.out.println("No fruit selected");
        }
    }
}

Java标准库提供了Arrays.toString()，可以快速打印数组内容 

import java.util.Arrays;

public class Main {

        int[] ns = { 1, 1, 2, 3, 5, 8 };
        System.out.println(Arrays.toString(ns));
    }
}

使用private 修饰class Person field 保证封装性
通过setter，getter方法访问并设置参数检查

  • 可变参数

用类型...定义，可变参数相当于数组类型 

class Group {
    private String[] names;//返回值为String

    public void setNames(String... names) {//返回值为void
        this.names = names;
    }
}

…可换成[]，但调用方需要自己构造String[]，另一个是调用方可以传入null

  • 参数绑定
  1. String 不可变
  2. 数组，引用内容可变

方法

参数绑定：
调用方把参数传递给实例方法时，调用时传递的值会按参数位置一一绑定。
基本数据类型参数绑定是调用方值的复制。
引用类型参数的传递，调用方的变量，和接收方的参数变量，指向的是同一个对象。

  • 构造方法

由于构造方法是如此特殊，所以构造方法的名称就是类名。构造方法的参数没有限制，在方法内部，也可以编写任意语句。但是，和普通方法相比，构造方法没有返回值（也没有void），调用构造方法，必须用new操作符。 

Person ming = new Person();
ming.setName("小明");
ming.setAge(12);

使用构造方法在创建对象的同时赋初值(初始化实例)
一个构造方法可以调用其他构造方法，语法：this(...) 

class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public Person(String name) {
        this(name, 18); // 调用另一个构造方法Person(String, int)
    }

    public Person() {
        this("Unnamed"); // 调用另一个构造方法Person(String)
    }
}
  • 方法重载(overload)

方法重载：方法名相同，参数列表不同.(返回值类型通常是相同的) 

public void setName(String name) {
        this.name = name;
    }
public void setName(String firstName,String lastName){
    this.name=firstName;
    this.name=lastName;
    }

继承

子类获得父类的所有字段，严禁定义与父类同名的字段
由于private修饰的字段无法被子类访问，削弱了继承的作用，用protected修饰的字段可以被子类访问
子类引用父类的字段用super.filedName
类似无参构造，java中任何class的构造方法，第一行语句必须是调用父类的构造方法，编译器会自动添加super();调用
如果父类没有默认的构造方法，子类就必须显式调用super()并给出参数以便让编译器定位到父类的一个合适的构造方法。

  • 向上转型

把一个子类类型安全地变为父类类型的赋值，称为向上转型(upcasting)
向上转型实际上是把一个子类类型安全地变为更加抽象的父类型 

Student s = new Student();
Person p = s; // upcasting, ok
Object o1 = p; // upcasting, ok
Object o2 = s; // upcasting, ok

注：Student继承自Person

向上转型有什么用处？=>多态

多态

多态：针对某个类型的方法调用，其真正执行的方法取决于运行时期实际类型的方法
多态的特性：运行期才能动态决定调用的子类方法

在继承关系中，子类如果定义了一个与父类方法签名完全相同的方法，被称为覆写(Override) 

public class Main {
    public static void main(String[] args) {
    }
}

class Person {
    public void run() {}
}

public class Student extends Person {
    @Override // Compile error!
    public void run(String s) {}
}

加上@override让编译器帮助检查
用final 修饰的方法不允许被覆写(Override)、extends

抽象类

如果父类的方法本身不需要实现任何功能，仅仅是为了定义方法签名，目的是让子类去覆写它，那么把父类的方法声明为抽象方法
抽象方法的本质是定义接口规范：规定高层类的接口 

abstract class Person {
    public abstract void run();
}

使用abstract 修饰，无法执行抽象方法的类称为抽象类，抽象类无法实例化。

接口

所谓interface就是比抽象类还要抽象的纯抽象接口，接口定义的所有方法默认public abstract
接口不能有实例字段，但可以有用final修饰的静态字段
简写： 

public interface Person {
    // 编译器会自动加上public statc final:
    int MALE = 1;
    int FEMALE = 2;
}

在接口中可以定义default方法

静态字段和静态方法

在一个class中定义的字段称为实例字段。实例字段特点：每个实例有独立的字段，各个实例的同名字段互不影响。
static filed 静态字段，只有一个共享空间，所有实例共享该空间。
类名.静态字段访问静态对象
调用实例方法需要实例变量，而调用静态方法通过类名就可以调用，类似其他语言的函数。
静态方法常用于工具类：

  • Math.random()
  • Arrays.sort()

静态方法也常用于辅助方法：main()

包

用来解决命名冲突，一个类总属于一个包
包作用域：不用public、protected、private修饰的字段和方法就是包作用域
位于同一个包的类，可以访问包作用域的字段和方法

作用域

嵌套类拥有访问private权限
定义为protected的字段和方法可以被子类访问，以及子类的子类
一个[.java]文件只能包含一个public类，但可以包含多个非public类。如果有public类，文件名必须和public类的名字相同。

定义方法：

  1. outer class 中明确指定

  2. 通过匿名类定义

  3. 静态内部类

classpath:指示JVM如何搜索class
jar:把目录打包
